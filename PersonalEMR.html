The increasing need to integrate data acquisition and analysis in personal health records (PHR) with Big Data reference such as The Cancer Genome Atlas (TCGA) comes with very specific requirements for bioinformatics application development. On the one hand the relevant TCGA cohort will be different between different patients. On the other hand the patient data itself should not escape control of the PHR that hosts it. This is sometimes described as a "beyond the data deluge" problem which finds a solution by having the code, rather than the data, travel between data domains. The modern web browser, with its high performance compiler and generous HTML5 APIs, backed by cloud computing resources, orchestrated by a web of linked data view of the semantic Web, are the key enablers of the use of the third generation of Web technologies to assemble computational constructs that are physically distributed while being logically whole. This reversal of the current service oriented architecture where it is the data that travels between analytical domains will be discussed, and will be illustrated with examples from cancer genomics integrated with personal health records (PHR).
